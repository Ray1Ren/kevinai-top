# 同题任务：做一款真正能玩的原创弹弓物理游戏

【设计意图】

做一款让普通玩家一眼能理解“拖弹弓、放手发射、撞塌建筑”的原创网页小游戏。核心乐趣可以让人联想到经典弹弓物理游戏，但角色、名称、画面、声音和关卡必须原创，不能照搬任何现有 IP。

动手前先用一句话复述你理解的任务边界，然后直接继续完成，不要等待回复。

【工作目录与必读文件】

- 只在当前目录工作。
- 当前目录里的 `SPEC.md` 就是本任务书，必须完整遵守。
- 工作区里出现其他文件或改动是正常的；不要读取、修改或回滚父目录、兄弟目录及其他 Agent 的产物。

【任务】

1. 制作一款名为 `Sling Siege / 弹弓攻城` 的单页网页游戏。
2. 只能使用原生 HTML、CSS、JavaScript、Canvas 与 Web Audio；不安装依赖，不联网，不请求任何外部资源。
3. 交付至少包括 `index.html`、`styles.css`、`game.js`。双击或通过本地 HTTP server 打开后即可玩。
4. 核心玩法必须完整：
   - 左侧有清楚可见的弹弓和待发射物；右侧有原创守卫、木/石结构与地形。
   - 鼠标或触摸拖拽发射物进行蓄力和瞄准，拖拽时必须出现清楚的力度/轨迹反馈；松手后按重力飞行。
   - 发射物、地面、结构块和守卫之间要有可感知的碰撞；结构能受冲击、位移、倒塌或破坏，守卫能受伤/被击败。
   - 每次发射后，玩家可以点击、触摸或按 Space 触发一次空中能力，能力必须有明显视觉反馈。
   - 至少 3 个真正不同的关卡：结构布局、守卫数量或难度递增，不能只是换背景色。
   - 消灭全部守卫即过关；弹药耗尽且仍有守卫则失败；支持下一关、重玩本关和从头开始。
5. UI 与完成度：
   - 首屏要有原创标题、简短玩法说明和开始按钮。
   - 游戏内显示关卡、分数、剩余弹药；提供暂停/继续、重开、静音按钮。
   - 要有胜利、失败、关卡切换反馈，以及粒子、拖尾、命中反馈、适度屏幕震动和程序化音效。
   - 桌面 1280×720 附近要有完整构图；390×844 手机宽度不得横向溢出，触摸可玩。
   - 视觉风格自行决定，但不能是默认组件拼盘，也不能只做调试原型。
6. 手感与难度：
   - 第 1 关应让第一次玩的用户在 1–3 次尝试内理解并过关。
   - 第 2、3 关逐步增加判断或操作难度，但不能靠极小碰撞区或纯运气。
   - 发射、飞行、碰撞、回合结束之间节奏清楚，不能长时间空等或卡死。
7. 状态与稳定性：
   - 最高分和最高解锁关卡写入 `localStorage`。
   - 页面不得出现未处理异常、明显内存泄漏、重复事件绑定、发射后永久卡住等问题。
8. 提供统一测试接口 `window.__SLINGSHOT_TEST__`，至少包含：
   - `snapshot()`：返回可序列化对象，至少含 `phase`、`level`、`score`、`shotsLeft`、`projectile`、`targets`、`blocks`。
   - `start()`、`restart()`、`loadLevel(level)`、`pause()`、`resume()`。
   - `setManualClock(enabled)` 与 `step(ms)`，用于可重复推进物理模拟；暂停状态下 `step(ms)` 不得推进游戏状态。
   - `aim(dx, dy)`、`launch()`、`activateAbility()`，用于走完整发射流程。
   - `forceHit(targetId)`，用于可重复验证守卫受击、得分和胜负状态。
9. 完成后自行运行必要检查；可以写临时测试文件，但交付目录不要留下浏览器 profile、录屏、依赖目录或无关文件。

【验收判据】

- `index.html`、`styles.css`、`game.js` 存在，页面可在本地 HTTP server 下直接打开。
- 无外部网络请求、无 console error、桌面与 390px 页面无水平溢出。
- 玩家能通过真实鼠标/触摸完成：开始 → 拖拽瞄准 → 松手发射 → 空中能力 → 命中/破坏 → 胜负 → 重玩/下一关。
- 三个关卡均可进入，布局有实质差异。
- `window.__SLINGSHOT_TEST__` 的规定方法存在，返回值可序列化，并能验证暂停、发射、命中、重开和关卡切换。

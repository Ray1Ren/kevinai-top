# 同题任务：做一款真正能玩的原创 3D 第一人称拆弹小游戏

【设计意图】

做一款让普通玩家一眼能理解“进入 3D 战术训练场、移动瞄准、与敌人交火、清场后拆除装置”的原创网页小游戏。核心体验可以让人联想到大家熟悉的第一人称战术射击与拆弹模式，但名称、地图、角色、画面、声音和代码必须原创，不能复刻任何现有游戏或 IP。

动手前先用一句话复述你理解的任务边界，然后直接继续完成，不要等待回复。

【工作目录与必读文件】

- 只在当前目录工作。
- 当前目录里的 `SPEC.md` 就是本任务书，必须完整遵守。
- 当前目录预置了 `vendor/three.min.js`：Three.js r147，MIT License，SHA-256 为 `f34446bf875b5fb0dcd93819ffe1d9e182d46634ee855f5d904c6c4ac7cdbc95`。可以直接用 `<script src="vendor/three.min.js"></script>` 获得全局 `THREE`。
- `vendor/three.min.js` 是三位参评者完全相同的唯一预置运行库；不要修改、替换或复制它。
- 工作区里出现其他文件或改动是正常的；不要读取、修改或回滚父目录、兄弟目录及其他 Agent 的产物。

【任务】

1. 制作一款名为 `Breach Point / 破门点` 的单页 3D 第一人称网页游戏，场景主题为原创的低多边形海港仓库拆弹训练场。
2. 只能使用原生 HTML、CSS、JavaScript、Canvas、Web Audio 与当前目录预置的 Three.js；不安装依赖，不联网，不请求任何外部资源。
3. 交付至少包括 `index.html`、`styles.css`、`game.js`。通过本地 HTTP server 打开后即可玩。
4. 必须是真 3D 第一人称体验：
   - 使用 `THREE.PerspectiveCamera` 与 `THREE.WebGLRenderer`，玩家能在 X/Z 平面移动并以第一人称视角观察场景。
   - 场景有可辨认的地面、天空/雾、灯光、墙体、仓库、集装箱或木箱、两条以上通路、至少一个高低层次，以及远处地标；不能只是几个盒子摆在空白平面上。
   - 墙体和主要掩体有碰撞，玩家不能直接穿过去或走出地图；相机不能掉到地面下。
   - 敌人、武器、装置和环境全部用程序化几何体/材质制作，不能用 2D 静态截图冒充 3D。
5. 桌面端真实操作：
   - 点击“开始任务”进入游戏；支持 Pointer Lock 鼠标视角，若浏览器拒绝 Pointer Lock，页面仍要提供可继续玩的拖拽视角降级。
   - `WASD` 移动，鼠标瞄准，左键射击，`R` 换弹，靠近装置后长按 `E` 拆除，`Esc` 或按钮暂停。
   - HUD 显示生命、弹匣/备弹、剩余敌人、倒计时、准星与当前目标；提供重开、暂停/继续、静音按钮。
6. 核心战斗与任务循环必须完整：
   - 至少 4 个原创低多边形敌人，分布在不同位置；敌人至少有巡逻、发现玩家、追击或寻找射击位置、攻击、受击、死亡这些可辨认状态。
   - 敌人不能全部原地不动；玩家也不能隔着实心墙直接击中敌人。允许用射线或简化碰撞实现视线与命中。
   - 玩家武器使用 12 发弹匣与至少 24 发备弹；必须有射速限制、换弹过程、空仓反馈，不能无限连点无代价。
   - 射击至少有枪口闪光、后坐/镜头反馈、弹道或曳光、命中火花、命中标记与音效；敌人攻击时有可感知的弹道/闪光、受伤红晕或方向提示。
   - 玩家生命归零、75 秒倒计时结束均判失败。消灭全部敌人后，玩家靠近发光装置并连续交互至少 1.5 秒才判胜利；中途松开要停止或回退进度。
   - 胜利/失败有明确结算层，显示用时、命中或得分信息，并可重玩。
7. 关卡与手感：
   - 开局 3 秒内必须通过场景构图、目标标记或短提示让第一次玩的用户知道往哪里走、要做什么。
   - 敌人威胁要能感知但不能开局瞬杀；普通玩家经过少量尝试应能完成一局。
   - 移动速度、鼠标灵敏度、射击反馈、敌人伤害和拆弹距离要合理，不能靠极小命中区、纯运气或等待很久。
   - 任务状态切换清楚，不能在清场、拆弹、胜负或重开阶段永久卡住。
8. 手机端：
   - 390×844 视口不得横向溢出，WebGL 画面保持可读。
   - 提供真实可用的触摸控制：左侧移动区、右侧拖拽观察，以及射击、换弹、交互按钮；不能只显示“请用电脑”。
9. 视觉与声音完成度：
   - 首屏有原创中英标题、简短玩法说明、操作提示和开始按钮，不能是默认组件拼盘。
   - 画面应有统一的低多边形美术方向、明确色彩层级、环境氛围、阴影或替代性的层次表现；敌我、可交互装置和掩体必须一眼可区分。
   - 使用 Web Audio 生成程序化枪声、命中、受伤、换弹、警报、胜负音效；默认音量适度并可静音。
   - 针对 `prefers-reduced-motion` 或低性能设备合理降低震动/粒子，但不能因此缺失主要反馈。
10. 状态与稳定性：
   - 最佳通关时间或最高分写入 `localStorage`。
   - 页面不得出现未处理异常、明显内存泄漏、重复事件绑定、WebGL 尺寸为 0、重开后多套循环叠加等问题。
   - 页面失焦或暂停时，AI、倒计时、射击冷却和拆弹进度不得继续推进。
11. 提供统一测试接口 `window.__BREACH_TEST__`，至少包含：
   - `snapshot()`：返回可序列化对象，至少含 `phase`、`paused`、`timeLeft`、`player`、`enemies`、`objective`、`stats`、`renderer`。其中 `player` 至少含 `x,y,z,yaw,pitch,hp,ammo,reserve,reloading`；`enemies` 每项至少含 `id,x,y,z,hp,state,alive`；`objective` 至少含 `state,progress,x,y,z`；`renderer` 至少含 `isWebGL,width,height,threeRevision`。
   - `start()`、`restart()`、`pause()`、`resume()`。
   - `setManualClock(enabled)` 与 `step(ms)`：用于可重复推进游戏；启用手动时钟后，真实 `requestAnimationFrame` 不得推进玩法状态；暂停状态下 `step(ms)` 也不得推进玩法状态。
   - `setPlayerPose({x,y,z,yaw,pitch})`：设置合法玩家位姿并同步相机。
   - `move(forward,right,ms)`：按玩家朝向尝试移动并遵守地图碰撞，两个方向值范围为 -1 到 1。
   - `aimAtEnemy(id)`、`shoot()`、`reload()`：走与真实输入相同的瞄准、射击、弹药和命中流程。
   - `damagePlayer(amount)`、`eliminateEnemy(id)`：用于可重复验证受伤、死亡、清场和任务阶段。
   - `interact(ms)`：只有满足真实距离和清场条件时才推进拆除进度。
12. 完成后自行运行必要检查；可以写临时测试文件，但交付目录不要留下浏览器 profile、录屏、依赖目录或无关文件。

【验收判据】

- `index.html`、`styles.css`、`game.js` 存在，页面可在本地 HTTP server 下直接打开。
- 无外部网络请求、无 console/page error，1280×720 桌面与 390×844 手机均无横向溢出。
- WebGL Canvas 占据主要可玩区域；场景、敌人、掩体、装置和第一人称移动在真实浏览器里可见。
- 玩家能通过真实鼠标键盘完成：开始 → 移动/观察 → 瞄准射击 → 受击/换弹 → 清场 → 靠近并长按拆除 → 胜利/失败 → 重玩。

- `window.__BREACH_TEST__` 的规定方法存在、返回值可序列化，并能统一验证手动时钟、暂停、地图碰撞、弹药、敌人状态、玩家死亡、清场、距离门槛、拆除胜利和重开。
